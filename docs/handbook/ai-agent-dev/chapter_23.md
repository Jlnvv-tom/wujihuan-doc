# 第23章 综合实战：基于 Coze + Dify 构建小红书爆款内容流水线

选题、文案、图片、发布，四个环节四个 Agent，串起来就是一条内容流水线。

我是怕浪猫，上一章做了 Manus + MCP 的全链路自动化，这章做另一个实战——用 Coze + Dify 构建小红书爆款内容流水线。

---

## 23.1 项目背景与目标

**行业痛点**

1. **选题难**：不知道写什么能火
2. **文案慢**：一篇小红书笔记写2小时
3. **图片累**：找图修图又花1小时
4. **发布烦**：手动排版发布容易出错

**项目目标**

构建一条自动化内容流水线：
1. **选题 Agent**：自动发现热门选题
2. **文案 Agent**：生成小红书风格文案
3. **图片 Agent**：生成配图或设计模板
4. **发布 Agent**：自动排版和发布

**技术选型**

- **Coze**：构建各个 Agent（工作流模式）
- **Dify**：知识库管理 + 文案润色
- **Coze 工作流**：串联各 Agent

---

## 23.2 选题 Agent：自动发现热门选题

**选题策略**

1. 热搜榜抓取（微博、抖音、小红书）
2. 关键词趋势分析
3. 竞品爆款内容分析
4. 季节性选题日历

**Coze 工作流设计**

```
[触发器] 定时/手动
    ↓
[搜索热门话题]
    ├── 抓取微博热搜
    ├── 抓取小红书热门笔记
    └── 抓取抖音热门话题
    ↓
[筛选相关话题] → 根据账号定位过滤
    ↓
[评估选题价值] → 综合评分
    ↓
[输出选题列表] → 按分数排序
```

**选题评估维度**

| 维度 | 权重 | 说明 |
|------|------|------|
| 热度 | 30% | 当前搜索量和讨论度 |
| 相关性 | 25% | 与账号定位的匹配度 |
| 时效性 | 20% | 是否是当前热点 |
| 竞争度 | 15% | 同类内容数量 |
| 可操作性 | 10% | 是否容易产出内容 |

**Coze 配置要点**

1. 搜索插件：配置搜索API（如 Serper）
2. 筛选逻辑：用 LLM 节点判断话题相关性
3. 评分逻辑：用代码节点计算综合评分
4. 输出格式：选题 + 评分 + 热度数据

## 23.3 文案 Agent：生成小红书风格文案

**小红书文案特点**

1. **标题党**：数字+情绪+好奇
2. **开头钩子**：3秒抓住注意力
3. **结构清晰**：分段+emoji+编号
4. **种草感**：个人体验>客观评价
5. **关键词**：SEO 友好的关键词布局

**Dify 知识库配置**

```
知识库内容：
1. 爆款文案模板库（50+ 篇爆款笔记拆解）
2. 行业关键词库
3. 账号风格指南
4. 禁用词列表
```

**文案生成工作流**

```
[输入选题]
    ↓
[检索知识库] → 获取相关爆款模板
    ↓
[生成标题] → 5个候选标题
    ↓
[生成正文] → 小红书风格
    ↓
[润色优化] → Dify润色节点
    ↓
[合规检查] → 检查禁用词
    ↓
[输出文案] → 标题+正文+标签
```

**标题生成提示词**

```
你是一个小红书爆款标题专家。根据以下选题生成5个标题：

选题：{topic}
账号定位：{account_position}
目标受众：{target_audience}

标题要求：
1. 包含数字（如"3个方法""5分钟"）
2. 触发情绪（好奇/焦虑/惊喜）
3. 20字以内
4. 不要标题党过度
5. 每个标题后标注使用的技巧

示例：
"3个习惯让我月入翻倍，第2个你一定没想到" → 数字+好奇
```

**正文生成提示词**

```
你是一个小红书文案写手。根据以下信息生成笔记正文：

标题：{title}
选题：{topic}
风格：个人体验+干货分享

要求：
1. 开头3行是钩子（制造好奇或共鸣）
2. 每段不超过3行
3. 合理使用emoji
4. 包含个人体验和具体数据
5. 结尾有互动引导
6. 300-500字
7. 底部加3-5个话题标签
```

## 23.4 图片 Agent：生成配图

**配图策略**

| 类型 | 说明 | 工具 |
|------|------|------|
| 封面图 | 吸引点击 | AI绘图 / Canva模板 |
| 内页图 | 配合正文 | AI绘图 / 图片搜索 |
| 信息图 | 数据可视化 | Canva / 代码生成 |

**图片生成工作流**

```
[输入文案内容]
    ↓
[分析图片需求] → 封面图+内页图数量
    ↓
[生成图片描述] → 详细的图片描述词
    ↓
[调用绘图API] → 生成图片
    ↓
[添加文字] → 封面加标题文字
    ↓
[输出图片集] → 按小红书尺寸裁剪
```

**封面图生成提示词**

```
生成小红书封面图的描述词：

标题：{title}
风格：{style}（如ins风/极简/可爱）
色调：{color_tone}

要求：
1. 3:4竖版比例
2. 背景简洁不杂乱
3. 留出标题文字空间
4. 色调统一和谐
```

## 23.5 发布 Agent：自动排版与发布

**发布工作流**

```
[接收文案+图片]
    ↓
[排版检查] → 检查换行、emoji、标签
    ↓
[定时设置] → 最佳发布时间
    ↓
[自动发布] → 调用小红书API
    ↓
[数据监控] → 发布后跟踪数据
```

**最佳发布时间**

| 时间段 | 说明 |
|--------|------|
| 7:00-9:00 | 早高峰通勤 |
| 12:00-13:30 | 午休 |
| 18:00-19:30 | 下班通勤 |
| 21:00-23:00 | 睡前浏览 |

**发布后数据监控**

```python
def track_post_performance(post_id):
    """跟踪发布后的数据表现"""
    
    # 每小时检查一次
    metrics = {
        "views": get_views(post_id),
        "likes": get_likes(post_id),
        "comments": get_comments(post_id),
        "shares": get_shares(post_id),
        "collects": get_collects(post_id)
    }
    
    # 计算互动率
    engagement_rate = (metrics["likes"] + metrics["comments"] + metrics["collects"]) / metrics["views"]
    
    # 判断是否爆款
    if metrics["views"] > 10000 and engagement_rate > 0.05:
        notify("这篇笔记可能要爆！")
    
    return metrics
```

## 23.6 完整流水线整合

**Coze 工作流串联**

```
[定时触发] 每天9:00和21:00
    ↓
[选题Agent] → 产出3个选题
    ↓
[人工确认] → 选1个（可选：自动选择最高分）
    ↓
[文案Agent] → 产出标题+正文+标签
    ↓
[图片Agent] → 产出封面图+内页图
    ↓
[发布Agent] → 定时发布
    ↓
[监控Agent] → 跟踪数据，反馈优化
```

**反馈闭环**

```
发布后数据
    ↓
分析表现好的内容特征
    ↓
更新选题策略
    ↓
更新文案模板
    ↓
下一轮产出质量更高
```

**成本估算**

| 环节 | 工具 | 单次成本 |
|------|------|---------|
| 选题 | Coze + 搜索API | ¥0.5 |
| 文案 | Dify + GPT-4o | ¥1.0 |
| 图片 | AI绘图API | ¥2.0 |
| 发布 | Coze自动化 | ¥0.2 |
| **合计** | | **¥3.7/篇** |

> 人工写一篇小红书笔记平均2小时，时薪50元的话就是100元/篇。AI流水线3.7元/篇，成本降低96%。但质量仍需人工把关，尤其是图片和品牌调性。
