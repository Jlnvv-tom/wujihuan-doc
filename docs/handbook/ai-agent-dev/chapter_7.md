## 7.1 从云服务到 AI 智能体的演进

**阿里云百炼是什么？**

阿里云百炼是阿里云推出的AI智能体构建平台，前身是阿里云的"通义千问"企业版。核心定位是"阿里云+钉钉+大模型"的一体化AI解决方案。

演进路径：

```
阿里云（基础设施）
    ↓
通义千问（大模型能力）
    ↓
阿里云百炼（智能体平台）
    ↓
钉钉集成（企业办公场景）
```

核心特性：

1. **钉钉原生集成**：直接发布到钉钉，作为AI助手使用
2. **阿里云深度集成**：调用阿里云各类服务（OSS、RDS、函数计算等）
3. **通义千问大模型**：内置阿里自研大模型
4. **数字人视频生成**：一键生成数字人视频
5. **AppFlow 自动化**：类似n8n的工作流编排

**和其他平台对比**

| 维度 | 阿里云百炼 | 腾讯元器 | Coze |
|------|-----------|---------|------|
| 钉钉集成 | 原生 | 无 | 无 |
| 阿里云集成 | 深度 | 无 | 无 |
| 数字人 | 支持 | 无 | 无 |
| 办公场景 | 强 | 中 | 弱 |
| 适合场景 | 钉钉办公 | 微信生态 | 内容创作 |

> 如果你的企业在用钉钉，阿里云百炼是唯一正解。就像腾讯元器之于微信，阿里云百炼是钉钉生态的天然选择。

## 7.3 实战：数字人视频生成

第一个实战，用阿里云百炼生成数字人视频——输入文字，自动生成带口型匹配的数字人视频。

**设计思路**

1. 输入要说的文字
2. 选择数字人形象和声音
3. AI 生成数字人视频
4. 下载或分享视频

**操作步骤**

1. 进入"数字人"页面
2. 选择数字人形象（有多种预设形象）
3. 输入要说的文字（支持多语言）
4. 选择声音（多种音色可选）
5. 点击"生成视频"
6. 等待生成完成（通常1-3分钟）
7. 预览和下载

**应用场景**

| 场景 | 说明 |
|------|------|
| 企业培训 | 生成培训视频，降低制作成本 |
| 产品介绍 | 生成产品介绍视频，用于营销 |
| 新闻播报 | 自动生成新闻播报视频 |
| 客服引导 | 生成客服引导视频 |

> 数字人视频的核心价值不是"替代真人"，而是"规模化内容生产"。一个真人一天录10条视频，数字人一天可以生成1000条。

## 7.4 实战：智能客服搭建

第二个实战，用阿里云百炼搭建智能客服系统。

**设计思路**

1. 上传产品文档和常见问题
2. 构建知识库
3. 创建智能客服 Agent
4. 集成到钉钉或网站

**知识库配置**

1. 进入"知识库"页面
2. 创建知识库，上传文档：
   - 产品说明书
   - 常见问题FAQ
   - 售后政策
3. 设置检索策略
4. 测试检索效果

**智能体配置**

```
人设：你是一个专业的客服助手，帮助用户解答关于[产品名]的问题。

工作流程：
1. 理解用户的问题
2. 在知识库中检索相关答案
3. 基于知识库内容回答
4. 如果知识库中没有相关内容，礼貌地告知用户并建议联系人工客服

注意事项：
- 只基于知识库内容回答
- 回答要简洁明了
- 态度要友好专业
```

**钉钉集成**

1. 在钉钉中创建机器人
2. 获取 Webhook URL
3. 在百炼中配置钉钉集成
4. 测试对话

## 7.6 实战：反钓鱼雷达

第四个实战，做一个反钓鱼邮件检测的智能体。

**设计思路**

1. 接收邮件内容
2. AI 分析邮件特征
3. 判断是否为钓鱼邮件
4. 给出风险评级和建议

**Agent 配置**

```
人设：你是一个网络安全专家，专门检测钓鱼邮件。

分析维度：
1. 发件人：域名是否可疑、是否模仿知名企业
2. 内容：是否有紧急催促、是否有可疑链接
3. 语法：是否有拼写错误、语法异常
4. 链接：链接域名是否和声称的企业一致

输出格式：
{
  "is_phishing": true/false,
  "risk_level": "高/中/低",
  "reasons": ["原因1", "原因2", ...],
  "suggestions": ["建议1", "建议2", ...]
}
```

## 7.7 实战：导购智能体

第五个实战，做一个电商导购智能体——根据用户需求推荐商品。

**设计思路**

1. 用户描述需求
2. AI 理解需求并提取关键参数
3. 调用商品搜索API
4. 生成个性化推荐

**Agent 配置**

```
人设：你是一个专业的导购助手，帮助用户找到最合适的商品。

工作流程：
1. 理解用户的需求（品类、预算、偏好等）
2. 提取关键搜索参数
3. 调用商品搜索工具
4. 根据需求筛选和排序
5. 生成推荐列表，包含理由

推荐格式：
1. [商品名] - [价格] - [推荐理由]
2. [商品名] - [价格] - [推荐理由]
...
```

## 7.8 实战：多智能体小说创作

第六个实战，用多个智能体协作创作小说——模仿 MetaGPT 的 SOP 驱动模式。

**设计思路**

用4个 Agent 协作完成小说创作：

1. **策划 Agent**：构思故事大纲和人物设定
2. **写作 Agent**：根据大纲撰写章节
3. **审校 Agent**：检查逻辑和文笔
4. **润色 Agent**：优化表达和情感

**工作流编排**

```
[主题输入] → [策划Agent] → [写作Agent] → [审校Agent] → [润色Agent] → [输出小说]
```

**4个 Agent 的人设**

策划 Agent：
```
你是一个小说策划专家。根据主题构思：
1. 故事大纲（3幕结构）
2. 主要人物设定（性格、动机、冲突）
3. 章节规划（每章核心事件）
输出完整的大纲文档。
```

写作 Agent：
```
你是一个小说作家。根据大纲撰写章节内容。
要求：
1. 遵循大纲的情节安排
2. 人物性格一致
3. 描写生动，对话自然
4. 每章2000-3000字
```

审校 Agent：
```
你是一个小说审校专家。检查：
1. 情节逻辑是否连贯
2. 人物性格是否一致
3. 有无错别字和语病
4. 节奏是否合理
输出修改建议列表。
```

润色 Agent：
```
你是一个文学润色专家。优化：
1. 描写的生动性
2. 对话的自然度
3. 情感的表达
4. 语言的节奏感
输出最终定稿。
```

> 多智能体协作的核心不是"人多力量大"，而是"专业分工"。每个 Agent 专注自己擅长的领域，组合起来完成复杂任务。

## 7.9 AppFlow 与 MCP 应用集成

**AppFlow 简介**

AppFlow 是阿里云百炼的工作流编排工具，类似 n8n 和 Dify 的工作流。

核心能力：

1. **可视化编排**：拖拽式设计工作流
2. **阿里云服务集成**：一键调用阿里云各类服务
3. **触发器**：定时触发、事件触发、API触发
4. **错误处理**：节点失败自动重试或通知

**MCP 应用集成**

MCP（Model Context Protocol）让智能体能调用外部工具和服务。

阿里云百炼的 MCP 集成：

1. 在"应用集成→MCP"中配置 MCP Server
2. 选择要使用的工具
