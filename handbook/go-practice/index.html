Go 实战训练营 - 知识结构目录 ​

课程概览 ​

课程定位 ​

针对 1-3 年初级 Go 开发工程师，系统提升实战能力，达到字节 2-1 水平

核心特色 ​

  • 大厂能力模型导向：直击开源框架及中间件底层核心原理
  • 实战驱动：手写 Web 框架、ORM 框架、缓存组件、微服务框架
  • 源码级教学：深入 Gin、Beego、Echo、Kratos 等主流框架实现
  • 面试导向：每个模块配备面试要点和常见问题

课程结构 ​

维度内容
周期15周
模块8大内容模块
实战4大实战案例
加餐4个热门面试主题
代码完整代码仓库

模块一：Web 框架（3周） ​

教学目标 ​

  1. 掌握 Web 框架的基本原理
  2. 掌握如何设计并实现一棵前缀路由树，支持通配符匹配、路径参数
  3. 掌握 HTTP Server 和 Context 的设计，并且提供丰富 API
  4. 掌握 HTTP 中 Session 的设计和实现
  5. 掌握 Web 框架中 AOP 的解决方案
  6. 设计并实现简单的静态资源服务器

学习痛点 ​

  • 用过很多 Web 框架，但不了解原理，不知道怎么注册路由、执行路由匹配
  • 面试时无法清晰阐述前缀路由树的原理，不知道如何快速定位 404 错误
  • 不知道如何设计统一的 Session 抽象，支持本地内存或 Redis
  • 无法灵活运用 AOP 方案解决登录校验、鉴权、日志、tracing、logging 等问题

第一周：Web 框架核心设计 ​

1.1 Web 框架概览与核心设计 ​

  • Web 框架架构演进：从标准库到框架化
  • 核心组件：Server、Context、路由树
  • 开源实例分析：Gin、Beego 中的 Server 和 Context 抽象

1.2 HTTP 包进阶语法 ​

  • net/http 包核心原理
  • Handler 接口设计与实现
  • 代码演示：Web 框架与 HTTP 包的结合点

1.3 前缀路由树设计与实现 ​

  • 开源实例：HttpRouter、Gin、Beego、Echo、Iris 路由树设计对比
  • 核心原理：前缀树（Trie）数据结构
  • 设计实现：
    • 路由节点结构设计
    • 通配符匹配实现（* 通配）
    • 路径参数提取（:param 语法）
  • 代码演示：实现前缀路由树，支持通配符匹配和路径参数
  • 作业：支持正则匹配路由
  • 面试要点：路由匹配算法复杂度分析

第二周：AOP 方案设计 ​

2.1 AOP 设计模式详解 ​

  • 责任链模式（Chain of Responsibility）
  • 洋葱模式（Onion Model）
  • 拦截器（Interceptor）机制

2.2 开源框架 AOP 实现对比 ​

  • Gin：Handler 链设计与实现
  • Beego：Filter 过滤器设计
  • Kratos：Middleware 中间件设计

2.3 代码演示：为 Web 框架提供 AOP 支持 ​

  • Access Log（访问日志）中间件
  • Tracing（链路追踪）中间件
  • Metric（指标监控）中间件
  • Recovery：从 panic 中恢复
  • 错误处理：返回特定错误页面或重定向
  • 面试要点：中间件执行顺序、panic 恢复机制

第三周：高级功能实现 ​

3.1 文件上传与下载 ​

  • 开源实例：Gin、Beego、Iris、Echo 文件处理对比
  • 文件上传：multipart/form-data 解析
  • 文件下载：Content-Disposition 处理
  • 进阶语法：Go 文件操作 API
  • 代码演示：实现文件上传下载功能

3.2 页面渲染与模板引擎 ​

  • 模板引擎设计原理
  • 进阶语法：Go text/template 与 html/template
  • 代码演示：在 Web 框架中支持页面渲染
  • 设计模式：Option 模式（函数式选项）
  • 代码演示：使用泛型设计通用的 Option 模式

3.3 静态资源支持 ​

  • 开源实例：各框架静态资源处理对比
  • 代码演示：设计可配置、可扩展的静态资源处理器
    • 缓存静态资源（内存缓存策略）
    • 控制缓存消耗（LRU 淘汰）
    • 大文件解决方案（分块传输）

3.4 Session 设计与实现 ​

  • 开源实例：Beego Session 模块、Gin Session 扩展
  • 代码演示：Session API 设计与实现
    • 基于内存的实现
    • 基于 Redis 的实现
  • 面试要点：Session 与 Cookie 区别、分布式 Session 方案

模块一实战项目 ​

项目名称：手写 Web 框架实现用户注册登录

项目内容：

  1. 设计并实现一个完整的 Web 框架
  2. 设计并实现前缀路由树（通配符、路径参数、正则匹配）
  3. 设计并实现 AOP 方案（logging、tracing、metric、panic 恢复、错误页面）
  4. 设计并实现静态资源服务器（缓存加速、内存控制）
  5. 提供文件上传下载、页面渲染功能
  6. 压测、分析、优化性能（CPU 和内存分配）
  7. 设计并实现 Session 机制
  8. 基于框架实现用户注册、登录服务
  9. 利用 AOP 接口实现用户级限流

教学目标 ​

  1. 掌握 ORM 框架的核心设计原理

5.3 事务管理 ​

  • 事务 ACID 特性
  • 事务传播行为

  • 代码演示：实现事务管理器（支持嵌套事务）

模块三：缓存系统（2周） ​

教学目标 ​

  1. 掌握缓存的核心设计原理与策略
  2. 掌握本地缓存（Local Cache）的实现
  3. 掌握 Redis 客户端的设计与实现

  5. 掌握缓存穿透、击穿、雪崩的应对策略

学习痛点 ​

第六周：本地缓存与 Redis 客户端 ​

6.1 缓存基础与设计原则 ​

  • 缓存适用场景与成本分析
  • 缓存设计原则：简单、高效、可控
  • 缓存淘汰策略：LRU、LFU、FIFO、TTL

6.2 本地缓存实现 ​

  • 开源实例：BigCache、FreeCache、ristretto 对比
  • 代码演示：实现线程安全的 LRU Cache
    • 双向链表 + Map 实现 O(1) 操作
    • 并发安全（分段锁或 sync.Map）
    • 内存控制与统计

6.3 Redis 协议与客户端设计 ​

  • Redis 协议（RESP）详解

第七周：缓存策略与一致性 ​

7.1 缓存模式 ​

  • Cache-Aside（旁路缓存）
  • Read-Through（读穿透）
  • Write-Through（写穿透）
  • Write-Behind（异步写回）

7.2 缓存问题与解决方案 ​

  • 缓存穿透：布隆过滤器、空值缓存
  • 缓存击穿：互斥锁、逻辑过期
  • 缓存雪崩：随机过期时间、多级缓存
  • 代码演示：实现缓存防护组件

7.3 缓存一致性 ​

  • 最终一致性 vs 强一致性
  • 延时双删策略
  • 消息队列保证一致性
  • 代码演示：实现缓存一致性管理器

  7. 性能优化：pprof 使用、内存分配优化、GMP 调度
